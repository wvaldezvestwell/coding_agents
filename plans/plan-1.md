# Implementation Plan — 1: Add GET endpoint to retrieve the list of USA states
> Auto-generated by /dev-start. Review before execution.

## Summary

Add a `GET /states` endpoint to the Web API project that returns the 50 USA states as a JSON array so client applications can populate state selectors and validate state input without maintaining their own copy of the list. The data is a hardcoded in-memory collection — no database, configuration file, or external service. The endpoint follows the existing minimal-API pattern already used by `/weatherforecast` in `Program.cs`.

## Scope

**In scope**
- `GET /states` endpoint in the Web API project (minimal API style).
- A `State` record with `Code` (2-letter USPS abbreviation, uppercase) and `Name`.
- All 50 states returned as a JSON array sorted alphabetically by `Name`.
- Endpoint registered with `.WithName("GetStates")` so it appears in the OpenAPI document.
- A static class holding the state list (`UsStates`) to keep `Program.cs` readable.

**Out of scope**
- Database or any persistence.
- Territories (PR, GU, etc.) and DC.
- Filtering, paging, or search.
- Authentication/authorization.
